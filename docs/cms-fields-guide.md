# 📝 Front Matter CMS: Повний гайд (Ultimate Longread)

Цей документ — ваша "біблія" по роботі з контентом на сайті Figarist. Тут описано все: від першого кліку, інструкції "для чайників", до тонких SEO-налаштувань та розбору кожної кнопки.

---

## 👩‍💻 Частина 1: Швидкий старт для новачків (Крок за кроком)

Цей розділ написаний максимально просто. Якщо ти вперше відкрила VS Code, щоб додати статтю чи урок на сайт Figarist — ти в правильному місці! 💜
Тобі не потрібно знати код. Весь процес автоматизований на 90%.

### Крок 1: Де створюється стаття?
1. В лівій частині екрану VS Code (на боковій панелі) знайди **іконку з аркушем паперу** (Front Matter CMS). Натисни на неї.
2. У меню, що відкриється зліва, натисни кнопку **"Create Content"** (Створити контент).
3. Вибери тип того, що ти пишеш:
   - **Post** — якщо це звичайна стаття у блог.
   - **Education** — якщо це навчальний матеріал чи урок.
4. Введіть назву файлу **англійськими літерами через дефіс** (наприклад: `my-first-post` або `how-to-draw-cat`). Це буде частина посилання на статтю в інтернеті. Натисни Enter.

### Крок 2: Заповнюємо головні поля 📝
Справа відкриється панель налаштувань. Тобі потрібно ОБОВ'ЯЗКОВО заповнити ці поля:
- **Title (Заголовок)**: Назва статті для людей (наприклад, "Як намалювати кота"). Це те, що буде великими літерами на сайті.
- **Description (Опис)**: Короткий опис про що стаття (1-2 речення, до 160 символів). Це той текст, який показує Google у пошуку під назвою сайту.
- **Language (Мова)**: За замовчуванням стоїть `uk` (українська). Якщо пишеш англійською — вибери `en`.
- **Author (Автор)**: Перевір, щоб тут стояло `ihor` (або вибери іншого автора з випадаючого меню, якщо потрібно).

### Крок 3: Магічні кнопки автоматизації 🪄
Не треба нічого прописувати вручну чи видумувати посилання! Проскроль панель налаштувань, що справа, в самий низ до розділу **"Custom Actions"** і по черзі натисни:
1. Кнопку **🔗 Auto Permalink** -> Вона сама створить та пропише правильне посилання для статті в поле Permalink.
2. Кнопку **🔄 Sync All Languages** -> Вона автоматично підготує файли-чернетки для інших мов сайту (англійської, російської, корейської) та зв'яже їх між собою одним ID. **Важливо: обов'язково тицяй цю кнопку після створення нової статті!**

### Крок 4: Картинка та Теги 🖼️
Повернися трохи вище в правій панелі налаштувань:
- Знайди блок **"Media"** і натисни на поле **Cover Image** (Головна картинка). Обери картинку для статті. *Супер-важливо: всі картинки мають бути у форматі `.webp`!*
- Обов'язково напиши **Image Alt** — що зображено на картинці (це потрібно для людей з вадами зору і для Google).
- Знайди блок **"Excerpts & Categories"** і вибери **Categories** (Категорію) та **Tags** (Теги), до яких відноситься стаття.

### Крок 5: Пишемо текст! ✍️
Тепер можна просто писати текст у лівому великому вікні едітора (під усіма налаштуваннями між `---` і `---`).
- Для заголовків у тексті став дві решітки та пробіл: `## Мій новий розділ`.
- Щоб зробити текст напівжирним, виділи його двома зірочками: `**жирний текст**`.
- Щоб вставити красиву картинку, відео чи щось інше — використовуй **Сніпети** (про них детально написано в Частині 4). Натисни `Ctrl` + `Пробіл` (або `{}` на панелі), щоб вибрати шаблон.

### Крок 6: Перевірка перед публікацією 📊
Знову проскроль в самий низ правої панелі і натисни:
- **📊 Validate SEO Meta** — це розумний скрипт, який перевірить, чи ти все заповнила правильно: чи не задовгий заголовок, чи нормальний опис, чи є картинка. Результат з'явиться в правому нижньому кутку. Якщо пише SEO SCORE: 100/100 зеленим — ти просто супер!

### Крок 7: Публікація 🚀
Коли стаття написана, вичитана і готова йти в світ:
1. Проскроль вгору правої панелі до самого першого блоку **PUBLISH CHANGES**.
2. У текстове поле **Commit message** напиши щось типу "Додано статтю про кота".
3. Натисни кнопку **Sync**.
Все! Стаття полетіла на сайт і з'явиться там за декілька хвилин.

## 📋 Частина 2: Повний довідник полів CMS

Для тих, хто хоче знати все "під капотом". Тут описано **кожен** елемент, який ви бачите в панелі CMS.

### 🔲 Секція: PUBLISH CHANGES

Ця секція знаходиться у самому верху панелі CMS та інтегрована з Git.

| Елемент | Опис | Коли використовувати |
| :--- | :--- | :--- |
| **Commit message** | Текстове поле для повідомлення коміту. | Введіть короткий опис змін перед синхронізацією. |
| **Sync** | Робить `git add` + `git commit` + `git push`. | Натискайте після збереження фінальних змін. |
| **Fetch** | Робить `git fetch` + `git pull`. | Натискайте, щоб отримати зміни інших. |

> 💡 **Tip:** Формат коміту автоматично задано в налаштуваннях як `content: {{title}} [{{date}}]`.

### 📑 Секція: METADATA (POST)

Основний блок метаданих — у ньому визначаються всі ключові властивості посту.

#### Головні поля

| Поле | Технічне ім'я | Тип | Обов'язкове | Опис |
| :--- | :--- | :--- | :--- | :--- |
| **Title** | `title` | `string` | ✅ | Головний заголовок сторінки (тег `<h1>`). Google обрізає після ~60 символів — стежте за лічильником `Field limit reached`. |
| **Description** | `description` | `string` | ✅ | Короткий опис для пошукових систем і соцмереж. Ідеальний розмір: 120–160 символів. Підказка під полем показує "`SEO meta description (~160 chars)`". |
| **Last Modified At** | `last_modified_at` | `datetime` | ❌ | Автоматично оновлюється через CMS. Використовується для `sitemap.xml` та поля `<lastmod>`. Кнопка **"now"** дозволяє встановити поточну дату/час. |
| **Date** | `date` | `datetime` | ✅ | Дата першої публікації посту. Визначає порядок в стрічках та URL (якщо вказано формат із датою). Кнопка **"now"** ставить поточну мітку. |
| **Language** | `lang` | `choice` | ✅ | Мова документа. Варіанти: `en`, `uk`, `ru`, `ko`. Визначає, який переклад інтерфейсу зі `_data/[lang]/strings.yml` підвантажується. |
| **Page ID** | `page_id` | `string` | ⚠️ | Унікальний ідентифікатор, що **зв'язує переклади**. Має бути **абсолютно однаковим** у файлах всіх 4 мов. Підказка: `Unique ID to link translated versions`. |
| **Permalink** | `permalink` | `string` | ❌ | Фінальний URL сторінки. Формат: `/blog/YYYY/MM/DD/slug/` або `/<lang>/blog/slug/`. Автоматично генерується кнопкою 🔗 **Auto Permalink**. |

| **Category** | `category` | `string` | Основний слаг категорії (напр. `gamedev`, `personal`, `tutorial`). Впливає на URL якщо layout використовує категорію. |
| **Categories** | `categories` | `categories` | Список всіх категорій Jekyll. Вибір із глобального списку таксономій. |
| **Tags** | `tags` | `tags` | Ключові слова для пошуку по сайту. Додавайте релевантні теги для фільтрації на дашборді та SEO. |

#### Поле: Author

Натисніть **"Add seo_group"** для розгортання.

| Поле | Технічне ім'я | Тип | За замовчуванням | Опис |
| :--- | :--- | :--- | :--- | :--- |

## ⚡ Частина 3: Секція ACTIONS — Вбудовані та кастомні кнопки

Кнопки дій знаходяться в нижній частині панелі CMS.

### Вбудовані кнопки Front Matter CMS

| Кнопка | Що робить |
| :--- | :--- |
| **Open dashboard** | Відкриває дашборд CMS у VS Code — там видно всі пости з фільтрами, сортуванням, пошуком. |
| **Optimize slug** | Генерує "чистий" URL-слаг із заголовка (прибирає спецсимволи, пробіли → дефіси, lowercase). |
| **Open preview** | Відкриває попередній перегляд сторінки у браузері (якщо налаштовано `previewPath`). |
| **Start server** | Запускає локальний сервер для розробки. |
| **Create content** | Створює новий файл посту/навчальної статті за вибраним шаблоном. |

### Кастомні кнопки (Custom Actions)

| # | Кнопка | Скрипт | Що робить | Коли використовувати |
| :--- | :--- | :--- | :--- | :--- |
| 1 | **🧹 Нормалізувати теги (Fix)** | `fix-frontmatter.js` | Прочитує поточний файл і його front matter, порівнює з JSON-схемою контент-типу (`Post`/`Education`), **видаляє** зайві поля, **додає** відсутні зі значеннями за замовчуванням. Виводить звіт: скільки полів збережено, додано, вилучено. | Після ручного редагування YAML або імпорту з іншого джерела. |
| 2 | **📊 Validate SEO Meta** | `check-seo.js` | Комплексна SEO-перевірка: **рахує SEO Score** (0–100), аналізує довжину title/description, перевіряє наявність та щільність keyword у тілі статті, формат permalink, наявність `lang`/`page_id`/`author`, структуру заголовків (H1 у тілі = помилка), внутрішні посилання. | **Перед кожною публікацією.** Найкорисніший скрипт. |
| 3 | **🔗 Auto Permalink** | `auto-permalink.js` | Генерує permalink із імені файлу: бере дату + слаг, додає мовний префікс якщо `lang != en`. Формат: `/blog/slug/` (en) або `/uk/blog/slug/` (uk). | Після створення нового файлу або зміни мови. |
| 4 | **🔄 Sync All Languages** | `sync-languages.js` | Синхронізує переклади: генерує `page_id` (якщо немає), оновлює permalink у всіх 4 мовних копіях, створює файли-заглушки для мов, яких ще немає. | Після створення контенту — для автогенерації перекладів. |
| 5 | **🏗️ Build Site (Manual)** | `build-manual.js` | Виводить у нотифікацію команду `bundle exec jekyll build`. Нагадування, бо CMS не може збирати Jekyll сама. | Коли потрібно перевірити збірку після змін у конфігурації. |
| 6 | **🌐 Create Missing Translations** | `create-translations.js` | Старий скрипт створення перекладів. **Рекомендовано використовувати 🔄 Sync All Languages замість нього.** | Легасі — для зворотної сумісності. |

> ⚠️ **Кнопка 🖼️ List Unoptimized Images** працює як `mediaFolder` action — вона з'являється на сторінці медіа-менеджера CMS, а не на сторінці посту.

## 💎 Частина 4: Сніпети (Snippets) та шорткоди

Сніпети вставляються через меню `Ctrl+Space` або іконку `{}`.

| Сніпет | Опис | Коли використовувати |

| **Spaceship: Figma Embed** | `!(https://www.figma.com/file/...)` | Ембед макету Figma. |
| **Spaceship: Local Video** | `!(/assets/images/...mp4)` | Для локальних відео (MP4). |
| **Spaceship: Mermaid Diagram** | Блок коду `mermaid` | Створення блок-схем, діаграм Ганта, UML. |
| **Spaceship: Math Block** | Блок `$$` | Математичні формули через MathJax. |
| **Spaceship: Markdown Table** | Звичайна таблиця | Базова Markdown-таблиця на 2 колонки. |
| **Callout: Info** | Робить красивий кольоровий блок для важливої інформації з іконкою 💡. |
| **Callout: Warning** | Робить червоний/жовтий блок-попередження (для помилок чи небезпек) ⚠️. |
| **Image: WebP with caption** | Тег `<figure>` + WebP | **Рекомендовано** для всіх картинок в тексті. |
| **Link: Internal post** | `[[title]](permalink)` | Правильна внутрішня перелінковка. |
| **Polyglot: Translation Note** | Рядок з прапорцями | Повідомлення про доступність мов (авто-згенеровано). |
| **Code: Rouge Highlight** | `{% highlight ... %}` | Підсвітка коду з номерами рядків. |
| **Code: Liquid Raw Block** | `{% raw %}` | Запобігає парсингу Liquid-тегів. |
| **Jekyll: Include partial** | `{% include ... %}` | Вставка Liquid-інклюду. |
| **SEO: Article Schema** | JSON-LD блок | Ручна розмітка Schema.org. |
| **Jekyll: Asset URL** | `{{ 'path' \| relative_url }}` | Коректне посилання на ассети через Liquid. |
| **Jekyll: Site String** | `{{ site.data[page.lang].strings.key }}` | Вставка локалізованого тексту зі `strings.yml`. |

## 💪 Частина 5: SEO & E-E-A-T Hacks

### 1. Довіра (E-E-A-T)
Google хоче знати, що статтю написав експерт. Ми використовуємо JSON-LD схему (автоматично) та вибір автора. Завжди перевіряйте правильність `author: ihor`.

### 2. Структура заголовків
Використовуйте `##` (H2) для розділів і `###` (H3) для підрозділів. **Ніколи** не використовуйте `#` (H1) всередині тексту — він зарезервований для назви статті з `title`.

### 3. Редіректи (Redirect From)
Якщо ви змінили назву чи категорію, старе посилання помре (404). Скопіюйте його в `redirect_from`, щоб користувачі (і Google) автоматично перейшли на нову адресу.

### 4. Focus Keyword
Правило трьох: ключове слово має бути **в заголовку**, **в описі** та **в тілі статті** (2–3 рази). Скрипт 📊 Validate SEO Meta рахує щільність автоматично.

### 5. Canonical URL
Якщо ви публікуєте один і той же контент на декількох платформах (dev.to, Medium), вставте URL оригіналу в `canonical_url` — інакше Google покарає за дублікати.
