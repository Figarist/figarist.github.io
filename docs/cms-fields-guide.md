# 📝 Front Matter CMS — Гайд по полях

Цей файл пояснює кожне поле, яке ти бачиш на панелі Front Matter CMS у VS Code.

---

## Основні поля

| Поле | Що це | Приклад |
|------|-------|---------|
| **Title** | Заголовок посту. Відображається на сайті як `<h1>` | `Як зробити гру в Unity` |
| **Description** | SEO мета-опис (~160 символів). Google показує це під заголовком у пошуку | `Покроковий гайд зі створення 2D гри для WearOS у Unity` |
| **Date** | Дата публікації. Використовується Jekyll для сортування постів | `2026-03-06` |
| **Language** | Мова посту (`en`/`uk`/`ru`/`ko`). Впливає на Permalink і Polyglot | `uk` |
| **Published** | Перемикач чернетка/публікація. `false` = чернетка (не з'явиться на сайті) | ✅/❌ |

## SEO та категоризація

### 🏷️ Tags (Теги)
**Що:** Ключові слова для фільтрації та пошуку на сайті.
**Навіщо:** Відображаються під постом, дозволяють фільтрувати на `/blog/`. Google використовує для розуміння контенту.
**Як:** Обирай з існуючих або набирай новий тег → Enter. Можна кілька.
**Приклади:** `unity`, `gamedev`, `csharp`, `life`, `wearos`, `vr`

### 📂 Category (Категорія)
**Що:** Основна рубрика посту. **Одна** на пост.
**Навіщо:** Визначає sidebar-навігацію та URL-ієрархію.
**Як:** Вписуєш slug-формат (латиниця, без пробілів).
**Приклади:** `gamedev`, `personal`, `tutorial`, `education`, `life`

### 📂 Categories (Категорії)
**Що:** Масив категорій для Jekyll. Може бути кілька.
**Навіщо:** Jekyll використовує для фільтрації на сайті. Відрізняється від Category тим, що це масив.
**Приклади:** `[unity, gamedev]`, `[life, personal]`

### ✍️ Excerpt (Витяг)
**Що:** Короткий опис (~100 символів) для карток на сторінках-лістингах (`/blog/`, головна).
**Навіщо:** Відображається під заголовком у картці посту. Якщо не вказано — Jekyll вирізає перший абзац.
**Приклад:** `Покроковий туторіал зі створення гри в Unity для WearOS`

## Ідентифікація

### 👤 Author (Автор)
**Що:** Автор посту. Обирається з `_data/authors.yml`.
**Навіщо:** E-E-A-T для Google, виводить ім'я та фото автора на сторінці посту.
**Як:** Обирай з дропдауну → `ihor`.

### 🔗 Permalink
**Що:** Постійна URL-адреса посту на сайті.
**Навіщо:** Визначає точний URL. Якщо не вказано, Jekyll згенерує з дати + slug.
**Як:** Натисни кнопку **🔗 Auto Permalink** в Actions (внизу панелі) — скрипт автоматично згенерує permalink з імені файлу.
**Формат:**
- EN → `/blog/how-to-make-a-game/`
- UK → `/uk/blog/how-to-make-a-game/`
- RU → `/ru/blog/how-to-make-a-game/`
- KO → `/ko/blog/how-to-make-a-game/`

### 🐌 Slug
**Що:** URL-дружня версія заголовку, без дати та спецсимволів.
**Навіщо:** Front Matter CMS використовує це для кнопки "Optimize Slug" — перетворює заголовок у формат `my-cool-post`.
**Як:** Натисни "Optimize Slug" або проігноруй — пермалінк все одно генерується з **імені файлу**, не зі slug.
> ⚠️ Slug — внутрішнє поле CMS. Jekyll його **не використовує**.

### 📖 TOC (Table of Contents)
**Що:** Перемикач, що вмикає або вимикає зміст статті (Table of Contents).
**Навіщо:** Якщо увімкнути (`toc: true`), на сторінці поста автоматично згенерується клікабельний зміст на основі заголовків `##` та `###`. Це дуже корисно для довгих лонгрідів!
**Як:** У панелі CMS це виглядає як галочка (перемикач `true/false`).

## Зображення

### 🖼️ Cover Image
**Що:** Головне зображення посту (og:image для соціальних мереж, лістинг).
**Як:** Використовуй Image Picker в CMS. Зображення має бути `.webp` в папці `assets/images/`.

---

## Типовий процес створення посту

1. **Content → Create content** → обрати "Post"
2. Заповнити **Title** і **Description**
3. Обрати **Language** (uk за замовчуванням)
4. Натиснути **🔗 Auto Permalink** в Actions
5. Обрати **Categories** (напр. `unity`, `gamedev`)
6. Додати **Tags** (напр. `unity`, `tutorial`, `csharp`)
7. Додати **Cover Image** (`.webp`)
8. Написати контент
9. Перемкнути **Published** → ✅
10. Git → Sync (кнопка в CMS панелі)

---

## Корисні кнопки (Actions)

| Кнопка | Що робить |
|--------|-----------|
| 🔗 Auto Permalink | Генерує permalink з імені файлу + мови |
| 🌐 Create Missing Translations | Створює файли-заглушки для інших мов |
| 🖼 List Unoptimized Images | Шукає не-webp зображення в `assets/images/` |
| Optimize Slug | Оптимізує slug з заголовку (CMS-поле, Jekyll не використовує) |
