# 📝 Front Matter CMS: Повний гайд (Ultimate Longread)

Цей документ — ваша "біблія" по роботі з контентом на сайті Figarist. Тут описано все: від першого кліку до тонких SEO-налаштувань та розбору кожної кнопки.

---

## 🚀 Частина 1: Швидкий старт для новачків

Ви щойно відкрили VS Code і хочете щось опублікувати. Не панікуйте. Весь процес автоматизований на 90%.

### 1️⃣ Створення файлу
В лівій панелі VS Code перейдіть на вкладку **Front Matter CMS** (іконка з аркушем).
*   Натисніть **"Create Content"**.
*   Виберіть **"Post"** (для блогу) або **"Education"** (для навчальних матеріалів).
*   Введіть назву файлу. **Краще використовувати англійські назви** (напр. `my-awesome-post`), бо це відразу формує частину URL.

### 2️⃣ Обов'язкові поля (The Must-Haves)
Як тільки файл створився, відкриється панель редагування справа. Заповніть:
*   `Title`: Заголовок, який побачать люди.
*   `Description`: Короткий опис (~155 символів). Це той текст, який Google показує під назвою сайту в пошуку.
*   `Author`: Завжди перевіряйте, щоб стояв `ihor`.

### 3️⃣ Магічні кнопки автоматизації
Прокрутіть панель CMS до самого низу. Там є блок "Custom Actions":
1.  **Натисніть 🔗 Auto Permalink**: Генерує URL з імені файлу.
2.  **Натисніть 🔄 Sync All Languages**: Створює копії для всіх мов та зв'язує їх одним `page_id`.

## 📋 Частина 2: Повний довідник полів (Exhaustive Reference)

Тут описано **кожен** елемент, який ви бачите в панелі CMS.

### 📑 Спільні поля (для всіх типів контенту)

| Поле | Технічне ім'я | Опис |
| :--- | :--- | :--- |
| **Title** | `title` | Головний заголовок сторінки (тег `<h1>`). Обов'язкове. |
| **Description** | `description` | Короткий опис для пошукових систем та соцмереж. |
| **Language** | `lang` | Поточна мова документа (`en`, `uk`, `ru`, `ko`). Визначає локалізацію інтерфейсу. |
| **Page ID** | `page_id` | **Ключове:** Унікальна стрічка, що зв'язує переклади. Має бути однаковою в усіх 4 файлах! |
| **Permalink** | `permalink` | Фінальний URL сторінки. Має починатися і закінчуватися на `/`. |
| **Author** | `author` | Вибір автора з `_data/authors.yml`. За замовчуванням `ihor`. |

---

### 📝 Специфічні поля: Блог (Post)

#### Блок: Categorization
- **Excerpt** (`excerpt`): Короткий вступ, що відображається на прев'ю картки.
- **Category** (`category`): Основний слаг категорії (напр. `gamedev`). Використовується в URL.
- **Categories** (`categories`): Список всіх категорій Jekyll.
- **Tags** (`tags`): Ключові слова для пошуку по сайту.

#### Блок: Media
- **Cover Image** (`image`): Основне зображення посту (вгорі). Тільки формат **.webp**.
- **Image Alt** (`image_alt`): Текстовий опис картинки для роботів Google та скрінрідерів.

#### Блок: SEO & Indexing
- **SEO Title** (`seo_title`): Можливість задати інший заголовок для вкладки браузера (якщо основний занадто довгий).
- **SEO Type** (`seo_type`): Тип розмітки Schema.org (`Article`, `BlogPosting` тощо). Впливає на те, як мобільно виглядає пост у Google.
- **Canonical URL** (`canonical_url`): Посилання на оригінальне джерело, якщо пост — передрук.
- **No Index** (`noindex`): Якщо увімкнено, Google не буде показувати цю сторінку в пошуку.
- **Sitemap** (`sitemap`): Чи додавати сторінку в карту сайту `sitemap.xml`.

#### Блок: Advanced
- **Redirect From** (`redirect_from`): Список старих URL для редіректу. Допомагає не ламати посилання при зміні слага.
- **Table of Contents** (`toc`): Чи генерувати зміст автоматично на основі заголовків.
- **Published** (`published`): Статус публікації. Якщо вимкнено — це чернетка.

---

### 🎓 Специфічні поля: Навчання (Education)

Тут поля схожі на Блог, але є важливі відмінності:
- **Sort Info / Level** (`level`): Рівень складності матеріалу (`beginner`, `intermediate`, `advanced`). Відображається як бейдж на картці.
- **Sort Info / Sort Order** (`sort_order`): Число для ручного сортування уроків у списку (від меншого до більшого).

## 💎 Частина 3: Сніпети (Snippets) та шорткоди

Сніпети вставляються через меню `Ctrl+Space` або іконку `{}`.

| Сніпет | Опис | Коли використовувати |
| :--- | :--- | :--- |
| **Spaceship: YouTube** | `!(https://youtube.com/...)` | Для вставки відео (адаптивно). |
| **Spaceship: Mermaid** | Блок коду `mermaid` | Створення блок-схем, діаграм Ганта, UML. |
| **Spaceship: Math** | Блок `$$` | Математичні формули через MathJax. |
| **Callout: Info/Warning** | HTML-бокс з іконкою | Для акцентів, порад або застережень. |
| **Image: WebP with caption** | Тег `<figure>` + WebP | **Рекомендовано** для всіх картинок в тексті. |
| **Link: Internal post** | `[[title]](permalink)` | Правильна внутрішня перелінковка. |
| **Polyglot: Translation Note** | Рядок з прапорцями | Повідомлення про доступність мов (авто-згенеровано). |
| **Code: Rouge Highlight** | `{% highlight ... %}` | Красива підсвітка коду з номерами рядків. |

## ⚙️ Частина 4: Кастомні скрипти (Custom Actions)

В самому низу панелі CMS є кнопки-скрипти:

1.  **🔗 Auto Permalink**: Бере назву файлу (напр. `2026-03-01-post.md`) і створює правильний слаг.
2.  **🔄 Sync All Languages**: "Маст-хев" для багатомовності. Копіює поточний файл у папки інших мов, зберігаючи метадані.
3.  **🏗️ Build Site (Manual)**: Якщо ви змінили щось у конфігурації Jekyll або хочете побачити зміни локально — вона виведе команду `bundle exec jekyll build` у ваш термінал.
4.  **🌐 Create Missing Translations**: Старий метод створення перекладів (використовуйте Sync замість нього).
5.  **🖼 List Unoptimized Images**: Аналізує папку `assets/images/` і каже, які файли не є `.webp` або занадто важкі.

## 💪 Частина 5: SEO & E-E-A-T Hacks

### 1. Довіра (E-E-A-T)
Google хоче знати, що статтю написав експерт. Ми використовуємо JSON-LD схему (автоматично) та вибір автора. Завжди перевіряйте правильність `author: ihor`.

### 2. Сруктура заголовків
Використовуйте `##` (h2) для розділів і `###` (h3) для підрозділів. Ніколи не використовуйте `h1` всередині тексту — він зарезервований для назви статті.

### 3. Редіректи (Redirect From)
Якщо ви змінили назву чи категорію, старе посилання помре (404). Скопіюйте його в `redirect_from`, щоб користувачі (і Google) автоматично перейшли на нову адресу.
